# Workflow 1: Project Setup (Solo Dev)

**When:** Once, Week 0
**Duration:** 30-60 minutes
**Output:** `docs/plan/project-plan.md`

---

## What to Do

Answer these 5 questions. That's your project plan.

### 1. What Are You Building? (5 min)
2-3 sentences answering:
- What's the product/app?
- What problem does it solve?
- Who uses it?

Example:
> MyRec is a macOS screen recording app for creators. Record your screen + audio with one click. Save as MP4. Essential for tutorial creators and content makers.

---

### 2. What's the Timeline? (5 min)
- How many weeks do you have?
- When does it ship?
- Any hard deadline?

Example:
> 6 weeks total. Ship MVP by end of week 4. Final release week 6.

---

### 3. Core Features (MVP) (5 min)
List 3-5 features that are MUST-HAVE:

Example:
- Screen recording (full screen)
- Audio capture (system + mic)
- MP4 output
- Stop recording button
- Save to ~/Documents

---

### 4. Success Definition (5 min)
How do you know it's done?

Example:
- Shipped and working
- Can record 1+ hour
- Build: 0 errors, 0 warnings
- All tests passing
- No crashes

---

### 5. Known Risks (5 min)
What could go wrong?

Example:
- Audio/video sync drift
- Memory usage on long recording
- macOS permission handling
- Performance on M1 Mac

---

## Just Fill the Template

Copy [template.md](./template.md) to `docs/plan/project-plan.md`

Fill in 5 sections. Done.

**No AI needed.** This is your vision.

---

## Checklist

- [ ] Overview complete (what + why)
- [ ] Timeline clear (weeks, deadline)
- [ ] MVP features listed (3-5 items)
- [ ] Success criteria written (5+ items)
- [ ] Known risks documented (3+ items)
